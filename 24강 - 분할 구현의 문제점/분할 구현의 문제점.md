# 헤더 파일에 함수를 구현하면 생기는 문제
`#include`로 헤더 파일을 불러오는 cpp파일이 여러개일 경우 모든 파일이 exe 파일로 합쳐지는 과정에서 `#include`의 복붙 특성 때문에
함수가 중복으로 복붙된다. 따라서 함수 중복 정의 오류가 발생한다.   
세부적인 설명으로는, 각 .cpp 파일이 컴파일될 때 해당 함수를 포함한 객체 파일(.o 또는 .obj 파일)이 만들어지는데,   
이 객체 파일들에 동일한 함수 구현이 중복으로 포함되는 문제이다.

# 헤더 파일에 전역 변수를 선언하면 생기는 문제
프로그램을 만들다 보면 cpp 파일이 여러개 생긴다.
이때 전역 변수 하나를 모든 파일에서 참조하고 싶을 때, 헤더 파일에 전역 변수를 선언하면 되지 않을까 생각을 해볼 수 있다.  
그러나 위에 함수와 마찬가지 이유로 모든 파일을 하나로 합쳤을 때, 변수가 중복으로 생기므로 오류가 발생한다.
