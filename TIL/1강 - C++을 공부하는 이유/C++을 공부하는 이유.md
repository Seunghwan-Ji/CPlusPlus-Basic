# 초창기 컴퓨터 언어
초창기 컴퓨터 언어는 기계어와 어셈블리어가 있었다.

## 기계어
컴퓨터는 전기적 신호로 이루어진 장치이다.
전류의 차단과 흐름을 0과 1인 2진수로 두가지의 상태 케이스로 정의한다.
이러한 2진수 개념의 데이터를 컴퓨터가 받아들이는 명령어를 기계어라고 한다.

## 어셈블리어
기계어의 각 이진수 명령어에 대해 사람이 이해할 수 있는 기호나 문자를 사용한다.
'어셈블러'라는 프로그램을 통해 기계어로 변환하여 프로그래머가 메모리 주소,
레지스터, CPU 명령어 등을 직접 제어할 수 있도록 도와준다.

초창기에는 이러한 명령어를 CPU 제조사마다 각기 다르게 외우고 있어야 했다.
그러나 운영체제의 등장으로 해당 운영체제에서 요구하는 명령어를 사용하면
제조사별 CPU에 맞는 명령어로 자동 변환되어서 편리해졌다.

# 고수준 프로그래밍 언어의 등장
어셈블리어 언어보다 다루기 쉬운 고수준 언어인 C, C++, C#, Java, Python 등이 생겨났다.
이 언어들은 사람이 생각하기 쉬운 직관적인 문법을 담고 있다.

C/C++ 언어는 '컴파일러'라는 것을 통해 기계어로 번역된다.

# 언어별 프로그램 실행 방식
C++ -> OS
Java -> 가상머신 -> OS
C# -> .NetFramework(가상머신) -> OS

C++은 가상머신을 거치지 않고 운영체제로 바로 실행이 되므로 속도가 가장 빠른 언어이다.
가상머신은 메모리 관리를 해주는 시스템이다.
그러므로 가상머신을 거치지 않는 C++은 메모리 관리를 수동으로 해줘야 한다.

속도, 최적화, 버그, 메모리 관리 등이 매우 중요한 게임 개발 분야에서는 C++ 언어가 필수이다.
