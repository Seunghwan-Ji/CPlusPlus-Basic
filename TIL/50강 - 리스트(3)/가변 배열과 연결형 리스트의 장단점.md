# 가변 배열
## 장점
1. 인덱스 접근 속도
- 배열은 메모리가 연속적으로 할당되어 있기 때문.
2. 메모리 효율성
- 요소들 사이에 추가적인 메모리(포인터)가 필요하지 않아,   
동일한 크기의 데이터를 저장할 때 더 적은 메모리를 사용.
3. 캐싱 효율
- 메모리 블록이 연속적이기 때문에 CPU 캐시 친화적, 데이터 읽기/쓰기 성능이 높음.
4. 직접 구현의 단순성
- 구현이 비교적 간단하며, 이미 많은 표준 라이브러리에서 제공.
## 단점
1. 삽입 및 삭제 속도
- 배열의 중간에 요소를 삽입하거나 삭제할 때, 요소들을 이동해야 하기 때문.
2. 크기 조정 비용
- 배열의 크기를 초과하면 새 메모리를 할당하고 기존 데이터를 복사해야 하기 때문.   
이로 인해 오버헤드가 발생.
3. 고정된 크기 문제
- 초기 크기를 초과하거나 너무 크게 설정하면, 메모리 낭비가 발생하거나 크기 조정에 따른 비용이 증가.

# 연결형 리스트
## 장점
1. 데이터 삽입 및 삭제 속도
- 포인터만 변경하면 되기 때문.
2. 동적 크기 조정
- 초기 크기를 설정할 필요 없이, 필요에 따라 메모리를 동적으로 할당,   
배열처럼 크기 초과 시 메모리 재할당이 필요하지 않음.
3. 메모리 효율성 (특정 상황)
- 데이터의 크기가 불규칙하거나 빈번한 삽입/삭제 작업이 필요한 경우,   
연결 리스트가 메모리 사용 측면에서 효율적일 수 있음.
## 단점
1. 인덱스 접근 속도
- 특정 요소에 접근하려면 처음부터 순차적으로 탐색해야 하기 때문.
2. 메모리 오버헤드
- 각 노드가 데이터와 함께 다음 노드를 가리키는 포인터를 저장하므로, 추가적인 메모리를 사용.
3. 캐싱 비효율
- 메모리가 불연속적으로 할당되므로, CPU 캐시 효율이 낮음.   
데이터 접근 속도가 배열보다 느림.
4. 구현의 복잡성
- 포인터를 관리해야 하므로 구현이 복잡하고,   
포인터 관련 오류(예: 메모리 누수)가 발생할 가능성이 있음.
