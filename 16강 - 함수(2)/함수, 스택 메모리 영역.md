# 함수가 동작하는 방식
스택 메모리 영역   
ex)   
함수3   
함수2   
함수1   
메인 함수   
   
메인 함수 -> 함수1 -> 함수2 -> 함수3 ->   
함수2 -> 함수1 -> 메인 함수 -> 프로그램 종료   
   
메인 함수가 먼저 호출되어 첫번째로 스택이 쌓인다.   
함수3은 가장 나중에 호출되어 스택이 쌓인다.   
가장 나중에 호출된 함수순으로 종료되면서 마지막에 가장 먼저 호출된 함수가 종료된다.   
따라서 선입 후출, 후입 선출 개념이다.   
   
`
int Add(int a, int b)
{
	return a + b;
}
`
   
`
int main()
{
	int iData = Add(100, 200);
	iData = Add(300, 400);
	iData = Add(222, 700);
	iData = Add(333, 555);
	return 0;
}
`   
1. main 함수 내 지역변수들의 데이터 크기를 더한 만큼의 스택 메모리 확보
2. 지역변수 iData에서 Add 함수 호출시 8바이트(a: 4Byte, b: 4Byte) 만큼의 스택 메모리 확보
3. Add 함수에서 값을 반환하면 Add 함수를 종료하고 확보한 8바이트의 스택 메모리 소멸
   
이런식으로 Add 함수가 호출될 때마다 임시적으로 8바이트의 스택 메모리를 확보한 후 함수가 종료되면 스택 메모리를 소멸시킨다.

# 함수 스택 메모리가 소멸하기전 return 값을 전달하는 과정
return 값 -> cpu 레지스터 메모리에 임시 저장 -> 함수 스택 메모리 소멸 -> 레지스터에 저장된 값을 함수를 호출한 변수에 전달
